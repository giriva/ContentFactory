# APPROVAL_WORKFLOW.md — You approve, the factory does the rest

## Where approvals happen
Every daily run commits a batch file to `content/pending/`. You get a GitHub notification (or a Gmail via Zapier).

## Your three moves (from your phone, <5 min/day)
1. **Approve** — move/rename the file into `content/approved/` (GitHub mobile: ... menu → move file), or comment "approve" on the commit.
2. **Approve with changes** — edit the file directly, then move it.
3. **Reject** — delete the file, optionally note why in `data/feedback.md` (the Researcher reads recent feedback).

## What never happens without you
- Publishing anything, anywhere
- Spending any money
- Changing niche or strategy
- Responding to comments/DMs as "you"

## What happens automatically
- Research, scriptwriting, production planning, weekly analysis, calendar proposals — all land as drafts and reports for your review.
